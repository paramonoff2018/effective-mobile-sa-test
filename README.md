# Схема отправки push-уведомлений

Файл `push_architecture.drawio.png` содержит схему архитектуры отправки push-уведомлений для мобильного приложения.

## Компоненты схемы

- **Сервисы-источники** (Order, Cart, Promo) — генерируют события
- **Брокер сообщений** (Kafka / RabbitMQ) — очередь событий
- **Notification Service** — центральный сервис уведомлений
- **User Preferences Service** — проверка настроек пользователя
- **FCM / APNS** — внешние шлюзы Google и Apple
- **Mobile App** — мобильное приложение

## Поток данных

1. Сервис-источник публикует событие в брокер
2. Notification Service забирает событие
3. Проверяет настройки пользователя
4. Формирует текст пуша
5. Отправляет в FCM/APNS
6. Пуш доставляется на телефон
